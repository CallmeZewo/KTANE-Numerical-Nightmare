On the Subject of Numerical Nightmare

Why isn’t this stupid thing working!?!?

  • The symbols on the right display update on each solve of a non-ignored module. The current stage is shown on the bottom display.
  • Each symbol has a designated value, listed in the table below, which is used to calculate the current stage’s number.
  • When all other non-ignored modules have been solved, the display will turn blank and wait for input.
  • A stage is valid if the following conditions are met:
    • The stage display shows the correct current stage (starting from 0).
    • Two symbols are shown on the right display.
  • For invalid stages, refer to “Fixing a module”.

Solving for stage value

If the digital root of the stage number is greater than 5:

Add the values of both symbols and the stage number.

If the stage number mod 5 is equal to 0:

Add the values of both symbols and the correct stage number of the last faulty stage. If there has been no faulty stage up to this point only use the symbols sum.

Otherwise:

Take the product of both symbols.

Take the least significant digit of your result.

Appendix M: Additional Notes

Here other things to keep in mind when making a manual page:

  • There are comments /* ... */ in the CSS code in the <style> blocks containing instructions. Follow those instructions before finishing your manual.
  • Normal and Needy module manual pages are formatted similarly. Widget manual pages are appendices and are formatted as such.
  • Each manual or appendix page should have a <div class="page-footer... at the bottom of the page, updated accordingly.
  • Don’t be afraid to ask for help if needed!
  • A few things to reiterate:
    • Make sure to never forget closing tags (the ones with a / in them).
    • For the most part, HTML versions of bolding and underlining is recommended.
    • Italics are not used in vanilla manuals besides flavor text and references to appendices, and therefore is not included above.
